# תבנית פרומפט למשימת קוד טובה

העתיקו, מלאו, ומחקו סעיפים שלא רלוונטיים.

```md
## הקשר

אנחנו עובדים על: [תיאור קצר של המוצר/המודול]
המצב הנוכחי: [מה קורה היום]
הבעיה: [מה לא עובד או מה חסר]

## מטרה

בצע/י את השינוי הבא בלבד:
[תיאור ממוקד של התוצאה הרצויה]

## לא מטרות

אל תעשה/י:
- [שינוי שלא רוצים]
- [קובץ/אזור שלא לגעת בו]
- [שינוי מוצרי שדורש אישור]

## קבצים או אזורים לבדיקה

בדוק/י במיוחד:
- `path/to/file`
- `path/to/folder`
- [זרימה או קומפוננטה רלוונטית]

## מגבלות

- אל תוסיף/י dependencies בלי הסבר ואישור.
- שמור/י על API קיים, אלא אם נאמר אחרת.
- שמור/י על טון וניסוח קיימים.
- העדף/י שינוי קטן וברור על ריפקטור רחב.

## פלט מצופה

בסיום, דווח/י:
- מה השתנה.
- באילו קבצים נגעת.
- אילו בדיקות הרצת.
- אילו סיכונים או שאלות נשארו.

## צ׳קליסט QA

- [ ] קראתי את ה-diff.
- [ ] בדקתי את הזרימה המרכזית ידנית או דרך בדיקה קיימת.
- [ ] אין שינוי scope לא מתוכנן.
- [ ] אין secrets, מידע אישי או לוגים רגישים.
- [ ] אין dependency חדש בלי הצדקה.

## הערות סיכון

שים/י לב במיוחד ל:
- [אבטחה/פרטיות]
- [סכמה/מיגרציה]
- [ביצועים]
- [UX/עברית/נגישות]
```

## דוגמה קצרה

```md
## הקשר

במסך ההרשמה, הטקסט מתחת לכפתור צפוף מדי ונשבר לא יפה במובייל.

## מטרה

תקן/י רק את הריווח והניסוח הקצר של הטקסט שמתחת לכפתור.

## לא מטרות

- לא לגעת ב-database או בסכמה.
- לא לשנות API או לוגיקה של הרשמה.
- לא להוסיף dependencies.
- לא לשכתב copy באזורים אחרים במסך.

## פלט מצופה

בסיום דווח/י:
- אילו קבצים השתנו.
- מה הסיכון, אם יש.
- אילו בדיקות או בדיקות ידניות ביצעת, כולל מובייל.
```
