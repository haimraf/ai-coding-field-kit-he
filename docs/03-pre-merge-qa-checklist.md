# צ׳קליסט QA לפני מיזוג קוד שנוצר בעזרת AI

לפני merge, עברו על הרשימה הקצרה הזאת.

## Build / Lint / Test

- [ ] הבנייה עוברת.
- [ ] lint עובר, אם קיים.
- [ ] בדיקות רלוונטיות רצות ועוברות.
- [ ] ברור מה לא נבדק ולמה.

## בדיקת scope

- [ ] השינוי תואם למשימה המקורית.
- [ ] אין קבצים או מודולים שנערכו בלי סיבה.
- [ ] אין ריפקטור גדול שמתחבא בתוך תיקון קטן.

## UX ותוכן

- [ ] הזרימה המרכזית נבדקה כמו שמשתמש אמיתי היה עושה אותה.
- [ ] טקסטים בעברית נשמעים טבעיים וברורים.
- [ ] אין שינויי copy לא נחוצים.

## אבטחה ופרטיות

- [ ] אין secrets, tokens או מידע אישי בקוד.
- [ ] אין לוגים שחושפים מידע רגיש.
- [ ] הרשאות, קלט משתמש ושגיאות נבדקו ברמה בסיסית.

## מידע וסכמה

- [ ] אין שינוי סכמה בלי מיגרציה ותוכנית rollback.
- [ ] אין שינוי לא מוסבר במבנה נתונים או API.
- [ ] ברור מה יקרה לנתונים קיימים.

## מובייל ו-Responsive

- [ ] אם יש UI, נבדק מסך קטן.
- [ ] אין שבירה בסיסית של פריסה, גלילה או כפתורים.

## מודעות ל-Rollback

- [ ] ברור איך לבטל את השינוי אם משהו נשבר.
- [ ] השינוי קטן מספיק כדי להבין ולהחזיר לאחור.
