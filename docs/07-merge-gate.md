# שער מיזוג ל-PR שנעזר ב-AI

שער המיזוג הוא בדיקה קצרה לפני שמכניסים שינוי לענף הראשי. הוא לא מחליף סקירת קוד, אבל עוזר לעצור שינויים רחבים מדי, לא בדוקים או לא מוסברים.

## לפני שמבקשים review

- [ ] ה-PR פותר משימה אחת ברורה.
- [ ] קראתי את ה-diff בפועל, לא רק את תקציר הכלי.
- [ ] אין קבצים שנוצרו בטעות או שוחזרו מענף ישן.
- [ ] אין dependencies, קבצי build או קונפיגורציה חדשים בלי סיבה מפורשת.
- [ ] אין secrets, tokens, מידע אישי או לוגים רגישים.
- [ ] הטקסט בעברית נקרא טבעי ולא כמו תרגום אוטומטי.

## בדיקות מינימום

- [ ] הרצתי בדיקות קיימות שמתאימות לשינוי.
- [ ] אם אין בדיקות אוטומטיות, ביצעתי בדיקה ידנית של הזרימה המרכזית.
- [ ] בדקתי קישורים פנימיים במסמכי Markdown ששיניתי.
- [ ] בדקתי שאין שגיאות whitespace בעזרת `git diff --check`.

## שאלות חוסמות

אם התשובה לאחת מהשאלות היא "לא בטוח/ה", לא ממזגים לפני בירור:

1. האם ברור למה השינוי נדרש?
2. האם ברור מה בדיוק השתנה?
3. האם ברור איך השינוי נבדק?
4. האם יש בעלות אנושית על ההחלטה למזג?

## תגובת review קצרה שאפשר להעתיק

```md
בדקתי את ה-PR מול שער המיזוג:

- Scope: [תקין/דורש צמצום]
- בדיקות: [מה הורץ]
- סיכונים: [אין/פירוט קצר]
- החלטה: [מאושר/נדרש תיקון לפני מיזוג]
```
